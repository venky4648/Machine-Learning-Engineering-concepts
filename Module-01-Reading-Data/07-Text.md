# 📑 Tab-Separated Values (TSV)

TSV files are similar to CSV, but they use tabs (`\t`) to separate columns.

```python
import pandas as pd

# Method 1: Using read_csv with sep parameter
df = pd.read_csv("data.tsv", sep="\t")

# Method 2: Using read_table (default separator is tab)
df = pd.read_table("data.tsv")
```

---

# 🔀 Custom Separators

Sometimes data is separated by pipes `|`, semicolons `;`, or even multiple spaces.

**Pipe-Separated:**
```python
df = pd.read_csv("data.txt", sep="|")
```

**Regex Separator:**
If the data is separated by an inconsistent number of spaces, use regular expressions.
```python
df = pd.read_csv("data.txt", sep=r"\s+")
```

---

# 📏 Fixed-Width Files (FWF)

Legacy mainframes often export Fixed-Width Files. There are no delimiters; instead, each column has a strictly defined character width.

Example:
```text
ID  Name      Age
101 Ram       25
102 John      30
```

Use `pd.read_fwf()`:

```python
df = pd.read_fwf("data.txt")
```

You can optionally define the exact widths using the `widths` parameter if Pandas cannot automatically infer them.

# 📝 Reading Raw Text

For Natural Language Processing (NLP) or log analysis, you might want to read a file where every row is just a string of text.

```python
with open("logs.txt", "r") as file:
    lines = file.readlines()

df = pd.DataFrame(lines, columns=["Raw_Log"])
# From here, you can use regex and string manipulation to parse the logs
```

---

# 📌 Summary

- Use `pd.read_table()` or `pd.read_csv(sep='\t')` for TSV files.
- Use `sep` with regex (e.g., `\s+`) for irregular spacing.
- Use `pd.read_fwf()` for legacy fixed-width data.
- Read as raw text lines for complex log parsing or NLP.
